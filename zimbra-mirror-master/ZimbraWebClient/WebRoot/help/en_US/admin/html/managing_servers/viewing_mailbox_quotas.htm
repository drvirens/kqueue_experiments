Viewing Mailbox Quotas

Account quota is the amount of space in megabytes that an account can use. The quota includes email messages, Calendar meeting information, task lists, files in Briefcase and RSS feed folders. When the quota is reached, all email messages are rejected and users cannot add files to their account. If you set the quota to 0, accounts do not have a quota.

Users will need to delete items off the server to get below their quota limit, or you can increase their quota.

You can check account mailbox quotas from a Domain's Mailbox Quota page. The Mailbox Quota displays the following information for each account sorted by mailbox size in descending order.  You can click on the Quota, Mailbox Size, or Quota Used headers to change the viewing order.

  • The Mailbox Size column shows the disk space used.

  • The Quota Used column shows what percentage of quota is used.

  • The Quota column shows the mailbox quota allocated to the account. Quotas are configured either in the COS or by individual accounts.

 

  • Note: From the ZCS CLI, you can configure the zimbraMailAllowReceiveButNotSendWhenOverQuota attribute to TRUE. When set to TRUE, a mailbox that exceeds its quota is still allowed to receive new mail and calendar invites.
