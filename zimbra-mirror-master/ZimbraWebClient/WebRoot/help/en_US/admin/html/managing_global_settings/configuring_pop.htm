Configuring POP

POP access can be enabled as a global setting or server setting. The server settings overrule the global settings.

Configure POP from Global Settings

  1. Go to Configure>Global Settings.

  2. Click POP in the Navigation pane.

  3. Select Enable POP3 services to allow POP connection to the Zimbra servers.

  4. Select Enable SSL for POP service to enable encrypted connections (HTTPS) or Enable clear text login to enable connects that are not encrypted (HTTP).

  5. Set the number of threads that should be available. The default is 100. The number set here is used for the server configuration of POP.

  6. Click Save.

  7. Restart the server in order to have these changes take effect. (CLI command zmcontrol restart)

Configure POP on Servers

  1. Go to Configure>Severs.

  2. Select the server to edit and in the gear icon click Edit.

  3. Click POP in the Navigation pane.

  4. Select Enable POP3 services to allow POP connection to the Zimbra servers.

  5. Select Enable SSL for POP service to enable encrypted connections (HTTPS) or Enable clear text login to enable connects that are not encrypted (HTTP).

  6. Set the advertised name to identify the server when a POP client connects. The default is the local server name.

  7. Set the address as the local IP address the POP service should bind to. This is the interface address on which POP3 servers should listen to.  If this is empty, it binds to all interfaces.

  8. Set the POP3 and POP3 SSL port numbers

  9. Click Save.

  10. Restart the server in order to have these changes take effect. (CLI command zmcontrol restart)
