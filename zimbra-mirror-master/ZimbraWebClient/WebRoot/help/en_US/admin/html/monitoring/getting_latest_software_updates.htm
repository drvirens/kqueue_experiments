Getting the Latest ZCS Software Updates

When ZCS is installed, the ZCS software update utility is automatically configured to check for the latest ZCS version once a day and if there is an update, to send notification to the address that is configured in the administration console’s Server Updates.

The default configuration is to check for updates daily and to automatically send a notification to the admin's mailbox when a new ZCS version is available. You can change these settings from the Tools and Migration>Software Updates page.

The dates and times ZCS checked for updates are saved to the Updates tab. When an update becomes available, an email notification is sent out daily until you update the ZCS version. If you do not want to receive an email notification of updates, uncheck the box for Send notification email when updates are available.

You can check for updates any time by clicking the Check now link.

Server Updates Settings

  • Server that checks for updates.  Only one ZCS server checks for updates. If the Check Now request for an immediate update is sent to a different server, the request is proxied to the server indicated in this field. The result of the update response is stored in LDAP and is accessible by all servers. The server that polls for updates must be able to make outgoing HTTP requests. Note: You may need to make special provisions in your firewall for that to happen.

  • Check for updates every. The default is to check every 1 day. A job is scheduled in the crontab to check for updates.  The cron job runs every 2 hours to check if the Check for Updates interval has passed.  If you wish to check for updates more often than every two hours, adjust the cron task for "/opt/zimbra/libexec/zmcheckversion".

  • Updates URL. This is the URL that the server connects to when checking for an update. The server transmits its ZCS version, platform and build number to this URL. When a server polls the URL,  the response is stored in the LDAP. The Updates tab displays the times that checks were made.

  • Send Notification Email To/From Addresses:  The Send notification email to address receives an email notification when an update is available. These addresses are configured to be the admin's email address, but you can change this.

  • Notification Subject and Message templates.  This is the template for the notification message that is sent.
