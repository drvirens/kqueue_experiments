Prevent Users from Changing Passwords

Normally, users can change their own passwords whenever they want.  If you enforce password change rules, users are forced to change the password on a regular basis. However, if you are using External Active Directory or External LDAP for authentication, you must prevent users from changing their passwords.  

Set Password locked to prevent passwords from being changed either at the COS level or per account.

To lock passwords for all users in a Class of Service

  1. Go to Configure>Class of Service.

  2. Select the COS and in the gear icon click Edit.

  3. In the Navigation pane, select Advanced.

  4. In the Password section, check the box Prevent user from changing password.

  5. Click Save.

To lock passwords for individual accounts

  1. Go to Manage>Accounts.

  2. Select the account and in the gear icon click Edit.

  3. In the Navigation pane, select Advanced.

  4. In the Password section, check the box Prevent user from changing password.

  5. Click Save.
